## 2. 従業員規模帯（Employee Band）

### 2-1. 定義

| コード | 名称 | 従業員数 |
|--------|------|----------|
| MICRO | 極小規模 | 1〜5名 |
| SMALL | 小規模 | 6〜20名 |
| SME_SMALL | 中小（小規模） | 21〜50名 |
| SME_MEDIUM | 中小（中規模） | 51〜100名 |
| SME_LARGE | 中小（大規模） | 101〜300名 |
| LARGE | 大企業 | 301名以上 |

### 2-2. 中小企業基本法に基づく定義

業種によって中小企業の定義が異なる。

| 業種 | 中小企業（従業員） | 中小企業（資本金） | 小規模事業者（従業員） |
|------|-------------------|-------------------|----------------------|
| 製造業・建設業・運輸業等 | 300人以下 | 3億円以下 | 20人以下 |
| 卸売業 | 100人以下 | 1億円以下 | 5人以下 |
| サービス業 | 100人以下 | 5,000万円以下 | 5人以下 |
| 小売業 | 50人以下 | 5,000万円以下 | 5人以下 |

### 2-3. API表記との対応

Jグランツ API の `target_number_of_employees` 表記パターン：

| API表記パターン | 解釈 |
|----------------|------|
| 中小企業・小規模事業者 | 業種に応じて判定（need_check） |
| 中小企業 | 業種に応じて判定（need_check） |
| 小規模事業者 | 業種に応じて5〜20人以下 |
| 50人以下 | 50人以下 |
| 100人以下 | 100人以下 |
| 300人以下 | 300人以下 |
| 全規模 | 制限なし |
| 大企業を除く | 中小企業のみ |

## 8. NG表現辞書（Prohibited Expressions）

申請書に記載すべきでない表現と、その理由・代替案。

### 8-1. 外販・販売関連

| NG表現 | 理由 | 代替案 |
|--------|------|--------|
| 将来的に販売 | 外販NGの補助金で致命的 | 「自社の業務効率化に活用」 |
| 横展開する | 補助対象外と判断される可能性 | 「自社内での展開」 |
| まずは自社で使い、将来 | 外販計画と見なされる | 「自社専用として継続利用」 |
| 他社に提供 | 外販と見なされる | 削除または表現変更 |
| ライセンス販売 | 外販と見なされる | 削除 |

### 8-2. 革新性・独自性関連

| NG表現 | 理由 | 代替案 |
|--------|------|--------|
| 業界標準として | 革新性がないと判断される | 「当社独自の取り組みとして」 |
| 一般的に | 革新性がないと判断される | 「当社では初めて」 |
| よくある | 革新性がないと判断される | 削除 |
| 普通の | 革新性がないと判断される | 「高機能な」「専用の」 |

### 8-3. 計画の曖昧さ関連

| NG表現 | 理由 | 代替案 |
|--------|------|--------|
| 一式 | 見積不明確と判断される | 具体的な内訳を記載 |
| 詳細は後日 | 計画不十分と判断される | 現時点で分かる範囲で記載 |
| 検討中 | 確定していないと判断される | 「予定」「計画」 |
| 未定 | 計画不十分と判断される | 仮でも具体的に記載 |
| 約○○ | 曖昧と判断される | 具体的な数値 |

### 8-4. 効果の不明確さ関連

| NG表現 | 理由 | 代替案 |
|--------|------|--------|
| 期待される | 確定していないと判断される | 「実現する」「達成する」 |
| 〜と思われる | 根拠がないと判断される | 「〜である」「〜となる」 |
| 可能性がある | 確定していないと判断される | 「〜する」 |
| 努力する | 確約がないと判断される | 「実施する」「達成する」 |

### 8-5. その他の注意表現

| NG表現 | 理由 | 代替案 |
|--------|------|--------|
| 補助金がないとできない | 自立性がないと判断される | 「補助金を活用することで加速」 |
| 赤字解消のため | 経営不安と判断される | 「収益性向上のため」 |
| 借入返済のため | 目的外使用と判断される | 削除 |
| 他の補助金でも申請 | 重複受給の懸念 | 削除または整理 |

## 10. Eligibility Operator

| Operator | 説明 | 例 |
|----------|------|-----|
| IN | 値が配列に含まれる | `{"operator": "IN", "value": ["東京都", "神奈川県"]}` |
| NOT_IN | 値が配列に含まれない | `{"operator": "NOT_IN", "value": ["風俗業"]}` |
| == | 等しい | `{"operator": "==", "value": true}` |
| != | 等しくない | `{"operator": "!=", "value": 0}` |
| < | 未満 | `{"operator": "<", "value": 300}` |
| <= | 以下 | `{"operator": "<=", "value": 300}` |
| > | 超過 | `{"operator": ">", "value": 0}` |
| >= | 以上 | `{"operator": ">=", "value": 1}` |
| BETWEEN | 範囲内 | `{"operator": "BETWEEN", "value": [1, 300]}` |
| CONTAINS | 配列に値が含まれる | `{"operator": "CONTAINS", "value": ["DX"]}` |
| NOT_CONTAINS | 配列に値が含まれない | `{"operator": "NOT_CONTAINS", "value": ["外販"]}` |
| EXISTS | 値が存在する | `{"operator": "EXISTS"}` |
| NOT_EXISTS | 値が存在しない | `{"operator": "NOT_EXISTS"}` |
| NOT_ONLY | 指定値のみで構成されていない | `{"operator": "NOT_ONLY", "value": ["汎用PC"]}` |
| MATCHES | 正規表現にマッチ | `{"operator": "MATCHES", "value": "^東京"}` |
