# 画面ワイヤー詳細仕様書

## 概要

本ドキュメントは、補助金マッチングシステムの各画面について、実装に必要な詳細仕様を定義します。

---

## 1. 画面一覧と遷移図

### 1-1. 画面一覧

| 画面ID | 画面名 | URL | 認証 | ロール |

## 2. 認証画面

### 2-1. AUTH-01: ログイン

#### レイアウト
```
┌────────────────────────────────────────────┐
│                  [ロゴ]                     │
│                                            │
│  ┌─────────────────────────────────────┐   │
│  │ メールアドレス                        │   │
│  │ [________________________]           │   │
│  └─────────────────────────────────────┘   │
│                                            │
│  ┌─────────────────────────────────────┐   │
│  │ パスワード                            │   │
│  │ [________________________] [👁]       │   │
│  └─────────────────────────────────────┘   │
│                                            │
│  [        ログイン        ]                │
│                                            │
│  パスワードをお忘れですか？                │
│  アカウントをお持ちでない方は こちら        │
│                                            │
└────────────────────────────────────────────┘
```

#### 入力項目

| 項目ID | 項目名 | 型 | 必須 | バリデーション |
|--------|--------|-----|------|----------------|
| email | メールアドレス | text | ○ | メール形式 |
| password | パスワード | password | ○ | 1文字以上 |

#### 状態

| 状態 | 条件 | 表示 |
|------|------|------|
| 初期 | 初回アクセス | 空フォーム |
| 入力エラー | バリデーション失敗 | フィールド下にエラー表示 |
| 認証失敗 | ログイン失敗 | 「メールアドレスまたはパスワードが正しくありません」 |
| 認証成功 | ログイン成功 | /companies にリダイレクト |
| 処理中 | API呼び出し中 | ボタン無効化 + スピナー |

#### API呼び出し

```typescript
// ログインボタン押下時
POST /auth/login
Request: { email: string, password: string }
Response: { access_token: string, refresh_token: string }
```

| hq_postal_code | 郵便番号 | text | ○ | 7桁数字（ハイフン任意） |

- 郵便番号入力後「住所を検索」で都道府県・市区町村を自動入力
